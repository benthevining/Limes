138 /* evaluation of 4 sines at once, using only SSE1+MMX intrinsics so
139  it runs also on old athlons XPs and the pentium III of your grandmother.
140 
141  The code is the exact rewriting of the cephes sinf function.
142  Precision is excellent as long as x < 8192 (I did not bother to
143  take into account the special handling they have for greater values
144  -- it does not return garbage for arguments over 8192, though, but
145  the extra precision is missing).
146 
147  Note that it is such that sinf((float)M_PI) = 8.74e-8, which is the
148  surprising but correct result.
149 
150  Performance is also surprisingly good, 1.33 times faster than the
151  macos vsinf SSE2 function, and 1.5 times faster than the
152  __vrs4_sinf of amd's ACML (which is only available in 64 bits). Not
153  too bad for an SSE1 function (with no special tuning) !
154  However the latter libraries probably have a much better handling of NaN,
155  Inf, denormalized and other special arguments..
156 
157  On my core 1 duo, the execution of this function takes approximately 95 cycles.
158 
159  From what I have observed on the experiments with Intel AMath lib, switching to an
160  SSE2 version would improve the perf by only 10%.
161 
162  Since it is based on SSE intrinsics, it has to be compiled at -O2 to
163  deliver full speed.
164 */
165 v4sf sin_ps (v4sf x);
