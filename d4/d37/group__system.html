◆ LIMES_ASSERT

#define LIMES_ASSERT (   condition)

#include <limes_assert.h>

Evaluates the condition, and if it is false, logs an assertion failure message and stops in the debugger if one is attached. This macro can be used in constexpr functions, but assertion failures will not be caught or logged if the function is evaluated at compile time. In release builds, this macro expands to nothing.

See also
LIMES_ASSERT_FALSE

◆ LIMES_ASSERT_FALSE

#define LIMES_ASSERT_FALSE

#include <limes_assert.h>

This macro, if reached within the control flow, always causes an assertion failure, and stops in the debugger if one is attached. This macro can be used in constexpr functions, but assertion failures will not be caught or logged if the function is evaluated at compile time. In release builds, this macro expands to nothing.

#define LIMES_COMPILER_WARNING (   text)

This macro causes the compiler to print the given text as a compilation warning, if available with the currently used compiler. Some compilers don't support user-defined warnings, so the text may just be printed to std::out during compilation with no actual warning issued.

◆ LIMES_CONSTEXPR_COPYABLE

#define LIMES_CONSTEXPR_COPYABLE (   ClassName)

#include <preprocessor.h>

Value:
\
constexpr ClassName (const ClassName&) = default; \ \

Similar to LIMES_DEFAULT_COPYABLE, this macro can be placed into a class definition to default its copy constructor and copy assignment operator, but they will also be declared constexpr. Example usage:

◆ LIMES_DISABLE_ALL_COMPILER_WARNINGS

#define LIMES_DISABLE_ALL_COMPILER_WARNINGS

#include <compiler_warnings.h>

This macro disables all compiler warnings from the point in the source code where it is encountered until LIMES_REENABLE_ALL_COMPILER_WARNINGS is encountered. This is useful for including external headers that may cause lots of warnings. Example usage:

LIMES_DISABLE_ALL_COMPILER_WARNINGS
#include <someExternalHeader>

◆ LIMES_UNREACHABLE

#define LIMES_UNREACHABLE

#include <limes_assert.h>

Invokes undefined behavior. The use case of this macro is to indicate to the compiler and to static analysis tools that a control path is intended to be unreachable. If a code path should not be reached, but may be in some cases, use LIMES_ASSERT_FALSE instead.
